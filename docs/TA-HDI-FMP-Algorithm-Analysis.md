# TA-HDI-FMP 改进计划与算法分析

## 1. 当前方案定位
你提出的 TA-HDI-FMP 主线是对的，核心价值在于把问题拆成四段：

1. 感知：检测示教轨迹与障碍物冲突区段
2. 探路：局部 RRT* 找可行绕障连通解
3. 插值：把稀疏节点转成高密度 via 点
4. 调制：用 FMP 将局部约束代数化并输出平滑轨迹

本质是“全局示教意图 + 局部拓扑避障”的组合框架。

## 3. 创新点评估
### 3.1 已有创新（可以写）
1. FMP 与 RRT* 的分层结合：示教意图与局部避障职责分离
2. HDI 高密度插值：缓解 RRT* 稀疏路径直接调制时的不稳定问题
3. 局部截断搜索：计算预算友好，适合在线场景

### 3.2 目前不足（需要补强）
1. 截断策略只看“首解”，缺少质量控制
2. HDI 固定步长，非自适应
3. FMP 调制里没有真正实现“协方差膨胀”的可控机制
4. 调制后没有闭环验收（碰撞/曲率/jerk 失败后的回退重规划）

## 4. 建议升级框架（RA-TA-HDI-FMP）
RA = Risk-Adaptive（风险自适应）

### 4.1 改进 A：预算内多解择优（替代首解即停）
在给定时间预算内收集少量可行解（例如 3 到 5 条），按以下代价挑最优：

`J = w1*长度 + w2*曲率 + w3*与示教偏差 + w4*障碍风险`

效果：实时性基本保留，轨迹质量显著提升。

### 4.2 改进 B：风险自适应 HDI
插值间距不固定，按局部风险和曲率调节：

- 近障碍 + 高曲率：加密
- 远障碍 + 低曲率：稀疏

效果：减少无效 via 点，提升关键区域约束效率。

### 4.3 改进 C：软硬混合约束调制
via 点拆分为两类：

- 硬约束点：确保安全边界不被突破
- 软约束点：维持平滑和意图，不强制逐点穿越

效果：降低“过约束引发回弹”的概率。

### 4.4 改进 D：协方差膨胀自适应化
不要全局固定一个 dilation，而是在调制中按局部风险动态调整：

- 风险高：增大感受域（增强绕障影响）
- 风险低：恢复原始参数（保留示教意图）

效果：统一处理安全与平滑矛盾。

### 4.5 改进 E：调制后闭环验收
输出轨迹后执行：

1. 连续碰撞检测
2. 曲率上限检测
3. jerk 上限检测

若不通过，只对失败片段触发局部重规划与重调制。
