# TA-HDI-FMP 仿真运行完整教程

## 目标
在 Ubuntu VM 中通过 ROS2 + Gazebo 仿真运行 TA-HDI-FMP 轨迹规划系统。

---

## 阶段一：环境确认

### 1.1 打开 VM 并登录

在 VM 中打开**终端**（快捷键 `Ctrl+Alt+T`）

### 1.2 检查 ROS2 环境

**基础原理**：ROS2 使用工作空间（workspace）来管理多个功能包，每个功能包包含节点、消息、服务等。

**执行命令**：
```bash
source /opt/ros/humble/setup.bash
ros2 --version
```

**预期输出**：
```
ros2 humile version X.X.X
```

**查看当前运行中的 ROS 进程**：
```bash
ros2 node list
ros2 topic list
```

如果报错说找不到 ros2 命令，说明需要重新 source 环境：
```bash
source /opt/ros/humble/setup.bash
```

---

## 阶段二：确认工作区

### 2.1 检查 ta_hdi_ws 工作区

**基础原理**：ROS2 工作区使用 `colcon` 编译工具，编译后需要 `source install/setup.bash` 来加载环境变量。

**执行命令**：
```bash
ls -la ~/ta_hdi_ws
```

**预期输出**：显示 `src/`、`build/`、`install/`、`log/` 目录（如果已编译）

### 2.2 如果工作区不存在（需要重新部署）

**基础原理**：由于是从 Windows 复制过来的，可能需要重新修复 BOM 和编译。

**执行步骤**：

1. **修复 BOM 问题**（Windows 复制过来的文件可能有 UTF-8 BOM 头）：
```bash
cd ~/ta_hdi_ws
find . -name "setup.cfg" -exec sed -i '1s/^\xef\xbb\xbf//' {} \;
find . \( -name "*.msg" -o -name "*.srv" \) -exec sed -i '1s/^\xef\xbb\xbf//' {} \;
```

2. **编译工作区**：
```bash
source /opt/ros/humble/setup.bash
colcon build --symlink-install
```

**预期输出**：
```
Summary: 4 packages finished [X.XXs]
```

如果出现错误，汇报给我。

## 阶段三：启动 Gazebo 仿真

### 3.1 启动 Gazebo TurtleBot3 世界

**基础原理**：
- **Gazebo** 是机器人仿真器，可以模拟物理环境、传感器、机器人模型
- **TurtleBot3** 是小型的轮式机器人平台，常用于学习和研究
- `turtlebot3_world.launch.py` 会启动 Gazebo 服务器和客户端，加载包含障碍物的仿真世界，并 spawn（生成）TurtleBot3 机器人

**环境变量说明**：
- `TURTLEBOT3_MODEL=waffle` - 指定机器人型号（Waffle 带有相机）
- `GAZEBO_MODEL_PATH` - 告诉 Gazebo 在哪里找机器人模型文件

**执行命令**：
```bash
# 设置环境变量
export TURTLEBOT3_MODEL=waffle
export GAZEBO_MODEL_PATH=/opt/ros/humble/share/turtlebot3_gazebo/models

# 启动仿真
ros2 launch turtlebot3_gazebo turtlebot3_world.launch.py
```

**预期**：会弹出一个 Gazebo 窗口，显示一个带有障碍物的房间和一个 TurtleBot3 机器人。

### 3.2 如果 Robot 没有出现（手动 spawn）

**基础原理**：`spawn_entity.py` 是 ROS2 提供的工具，可以向 Gazebo 发送请求来生成机器人模型到仿真世界中。

**打开第二个终端**，执行：
```bash
source /opt/ros/humble/setup.bash
ros2 run gazebo_ros spawn_entity.py -database turtlebot3_waffle -entity waffle -x -2.0 -y -0.5 -z 0.01
```

**预期输出**：
```
[INFO] [spawn_entity]: Spawn status: SpawnEntity: Successfully spawned entity [waffle]
```

## 阶段四：启动 TA-HDI-FMP 节点

### 4.1 打开新终端，加载工作区

**基础原理**：每次新终端都需要 source ROS2 和工作区的 setup.bash 来注册包路径。

**执行命令**：
```bash
source /opt/ros/humble/setup.bash
cd ~/ta_hdi_ws
source install/setup.bash
```

### 4.2 启动规划器节点

**基础原理**：
- **节点（Node）** 是 ROS2 中的基本执行单元
- `ta_hdi_fmp_planner` 节点提供 `/ta_hdi_fmp/plan_path` 服务，接收规划请求并返回路径

**执行命令**：
```bash
ros2 run ta_hdi_fmp_planner planner_node
```

**预期输出**：
```
[INFO] [ta_hdi_fmp_planner]: Planner service ready: /ta_hdi_fmp/plan_path
```

### 4.3 打开另一个终端，启动移动适配器节点

**基础原理**：
- `mobile_adapter` 节点订阅路径话题，执行路径跟踪，并向机器人发送速度命令

**执行命令**：
```bash
source /opt/ros/humble/setup.bash
source ~/ta_hdi_ws/install/setup.bash
ros2 run ta_hdi_fmp_mobile_adapter mobile_adapter_node
```

---

## 阶段五：启动 RViz 可视化

### 5.1 打开新终端，启动 RViz

**基础原理**：
- **RViz** 是 ROS2 的三维可视化工具
- 可以显示机器人模型、路径、传感器数据、地图等
- 需要设置 **Fixed Frame**（固定坐标系）才能正确显示

**执行命令**：
```bash
source /opt/ros/humble/setup.bash
rviz2
```

### 5.2 在 RViz 中添加显示项

**基础原理**：RViz 需要手动添加"显示（Display）"来查看不同类型的数据。

**手动操作步骤**：

1. **左下角 Displays 面板 → Global Options**
   - **Fixed Frame** 改为 `odom`（注意：不是 `map`，因为没有运行 SLAM）

2. **点击左下角 "Add" 按钮**，添加以下显示：

| 显示类型 | 名称 | Topic |
|----------|------|-------|
| Path | Planned Path | `/ta_hdi_fmp/planned_path` |
| MarkerArray | Obstacles | `/ta_hdi_fmp/debug_markers` |
| RobotModel | Robot | `/robot_description` |

3. **Path 显示属性**：
   - Color: 绿色 (0, 1, 0)
   - Alpha: 0.8
   - Line Width: 0.05

## 阶段六：发送规划请求

### 6.1 基础规划请求（无障碍物）

**基础原理**：
- `ros2 service call` 用于调用 ROS2 服务
- `/ta_hdi_fmp/plan_path` 服务的请求格式包含：
  - `start`: 起点位置 (x, y, theta)
  - `goal`: 目标位置 (x, y, theta)
  - `obstacles`: 障碍物列表

**执行命令**：
```bash
ros2 service call /ta_hdi_fmp/plan_path ta_hdi_fmp_msgs/srv/PlanPath2D '{
  start: {x: -2.0, y: -0.5, theta: 0.0},
  goal: {x: 2.0, y: -0.5, theta: 0.0},
  obstacles: []
}'
```

**预期响应**：
```
response:
  success: True
  status: OK
  min_dist: inf
  ...
```

### 6.2 带障碍物的规划请求

**执行命令**：
```bash
ros2 service call /ta_hdi_fmp/plan_path ta_hdi_fmp_msgs/srv/PlanPath2D '{
  start: {x: -2.0, y: -0.5, theta: 0.0},
  goal: {x: 2.0, y: 0.0, theta: 0.0},
  obstacles: [{type: 1, cx: 0.0, cy: 0.0, radius: 0.8, width: 0, height: 0, theta: 0}]
}'
```

**预期响应**：
```
response:
  success: True
  status: OK
  min_dist: 1.200
  kappa_p99: 0.495
  ...
```

**预期在 RViz 中**：看到绿色路径绕过红色障碍物

## 阶段八：查看坐标系变换

**基础原理**：
- **TF (Transform)** 是 ROS2 中表示坐标系之间关系的数据结构
- `/odom` 是里程计参考系
- `/base_link` 是机器人本体参考系

```bash
ros2 run tf2_ros tf2_echo odom base_link
```

```bash
ros2 run gazebo_ros spawn_entity.py -database turtlebot3_waffle -entity waffle -x -2.0 -y -0.5 -z 0.01
```

### 问题 3：RViz 中看不到机器人

```bash

ros2 service list | grep plan_path

# 检查节点是否运行
ros2 node list | grep planner
```
