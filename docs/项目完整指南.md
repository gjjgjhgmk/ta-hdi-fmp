## 1. 项目概述

### 1.1 什么是 TA-HDI-FMP？

TA-HDI-FMP（Task-Aware High-Dimensional Informed Rapidly-exploring Random Tree with Full Motion Planning）是一个分层混合轨迹规划框架，核心思想是：

> **"全局示教意图 + 局部拓扑避障"**

```
示教轨迹 ──► 检测冲突区段 ──► 局部 RRT* 绕障 ──► 自适应插值 ──► FMP 调制 ──► 输出平滑轨迹
    T1           T2              T3              T4             T5            T6-T7
```

### 1.2 解决的问题

| 场景 | 原始问题 | TA-HDI-FMP 方案 |
|------|----------|-----------------|
| 示教轨迹穿障 | 固定阈值裁剪不稳定 | RRT 局部重规划 |
| 稀疏路径调制震荡 | 直接插值约束不足 | HDI 自适应加密 |
| 安全与平滑矛盾 | 全局固定约束 | 软硬混合约束 |
| 计算资源限制 | 无限迭代 | 时间预算 + 候选池 |

## 3. 核心算法原理

### 3.1 T3: Informed RRT* 候选池

**原理**：标准 RRT* 的渐进最优性在时间预算内无法等待，所以引入"时间预算 + 质量容忍"机制。

```
算法流程：
1. 从 start 向 goal 扩展 RRT 树
2. 每当找到可行路径，检查 cost ≤ c_best × tolerance
3. 达到质量要求的路径加入候选池
4. 时间预算耗尽或迭代达上限停止
5. 按 cost 排序，保留 top-k 候选
```

**关键参数**：
- `time_budget_per_segment`: 时间预算（秒）
- `k_candidates`: 保留候选数量
- `rrt_quality_tolerance`: 1.15 表示允许 15% 的次优解

**去重机制**：
```python
if abs(p1.cost - p2.cost) < 1e-3 AND 中点距离 < 0.5:
    # 认为几何重复，跳过
```

### 3.2 T4: Adaptive HDI（自适应高密度插值）

**原理**：传统等距插值在转弯急、曲率大的地方约束不足。HDI 根据局部风险和曲率动态调整插值密度。

```
插值间距计算：
interp_dist = base_interp_dist / (1 + k_risk × risk_norm + k_curvature × kappa_norm)

其中：
- risk_norm = risk / (risk + 1)  # 风险越大，间距越小
- kappa_norm = kappa / ref       # 曲率越大，间距越小
- 实际间距被限制在 [min_interp_dist, max_interp_dist]
```

**危险区标记**：
```python
is_danger = min_point_to_obstacles(point) < safe_margin
```

### 3.3 T5: FMP 调制

FMP（Full Motion Planning）的核心是通过模糊聚类和模糊回归，将局部几何约束转化为代数调制信号。

**不修改原则**：`func/` 目录下的 MATLAB 函数是底层实现，主脚本只调用不修改。

### 3.4 T6: 路径评分

```python
J_total = w1 × J_length + w2 × J_curvature + w3 × J_risk + w4 × J_demo

其中：
- J_length   = path_length / ref_length
- J_curvature = kappa_max / kappa_thresh
- J_risk     = 0.7/(min_dist+0.1) + 0.3/(mean_dist+0.1)  # 混合风险
- J_demo     = shape_distance_to_reference
```

### 3.5 T7: 轨迹验收

三个硬约束必须全部通过：

| 指标 | 计算 | 通过条件 |
|------|------|----------|
| `min_dist` | 所有点到障碍物的最小距离 | > safe_margin |
| `kappa_p99` | 曲率的 99 百分位 | < kappa_pass_thresh |
| `jerk_rms` | 加加速度 RMS | < jerk_pass_thresh |

## 7. 核心模块详解

### 7.1 planner_api.py（统一入口）

```python
from ta_hdi_fmp_core import plan_path, PlannerConfig

# 使用默认配置
result = plan_path(start_xy, goal_xy, obstacles)

# 使用自定义配置
cfg = PlannerConfig()
cfg.time_budget_per_segment = 5.0
cfg.safe_margin = 0.5

# 返回值
{
    'success': bool,           # 是否通过验收
    'path_xy': np.ndarray,     # 路径点 (2, N)
    'metrics': {
        'min_dist': float,     # 最小障碍距离

        'jerk_rms': float,     # jerk RMS
        'path_length': float,  # 路径长度
    },
    'debug': {
        'candidate_used': int, # 使用的候选索引
        'n_via': int,          # via 点数量
        'status': str,         # 'OK' / 'NO_CANDIDATE' / 'VERIFY_FAIL'
    }
}
```

### 7.2 rrt_pool.py（Informed RRT*）

```python
@dataclass
class PathCandidate:
    path: np.ndarray      # 路径 (2, N)
    cost: float           # 路径代价
    t_found: float        # 找到时间
    iter_found: int       # 找到时的迭代次数
```

**碰撞检测**：
- 圆形障碍物：距离判断 `norm(point - center) ≤ radius + inflation`
- OBB 障碍物：坐标变换到局部框，再做 AABB 检测

### 7.3 hdi.py（自适应插值）

**输入**：RRT 稀疏路径 (2, N)
**输出**：
- `dense_path`: 加密后的路径点 (2, M)
- `t_local`: 对应时间序列
- `is_danger_mask`: 每点的危险标记

### 7.4 geometry.py（几何计算）

```python
def min_point_to_obstacles(point: np.ndarray, obstacles: np.ndarray) -> float:
    """计算点到所有障碍物的最小距离"""

def point_to_obstacle_distance(point: np.ndarray, obs: np.ndarray) -> float:
    """计算点到单个障碍物的距离"""
```

```python
def compute_discrete_curvature(path: np.ndarray) -> np.ndarray:
```

## 8. 配置参数说明

### 8.1 PlannerConfig 默认值（已修改版）

```python
@dataclass
class PlannerConfig:
    # ===== 示教处理 =====
    demo_len: int = 200              # 示教轨迹点数
    demo_dt: float = 0.1            # 示教时间步长
    alpha: float = 0.1              # 模糊聚类参数

    # ===== 环境生成 =====
    num_obs_target: int = 40        # 目标障碍物数量
    min_gap: float = 1.0            # 障碍物最小间距
    safe_margin: float = 0.5        # 安全边界（已从1.5改为0.5）

    # ===== RRT* =====
    step_size: float = 1.5          # 步长
    r_neighbor: float = 4.0        # 邻居半径
    max_iter_cap: int = 8000        # 最大迭代次数
    time_budget_per_segment: float = 5.0  # 时间预算（已从0.3改为5.0）
    goal_sample_prob: float = 0.2   # 目标采样概率
    k_candidates: int = 3           # 保留候选数量
    rrt_quality_tolerance: float = 1.15  # 质量容忍度

    # ===== HDI =====
    base_interp_dist: float = 0.25 # 基础插值间距
    min_interp_dist: float = 0.05   # 最小插值间距
    max_interp_dist: float = 0.6    # 最大插值间距
    hdi_k_risk: float = 2.0         # 风险驱动系数
    hdi_k_curvature: float = 1.5    # 曲率驱动系数

    # ===== 评分 =====
    score_weights: tuple = (0.4, 0.2, 0.3, 0.1)  # (长度, 曲率, 风险, 偏离)

    # ===== 验收 =====
    kappa_pass_thresh: float = 2.0  # 曲率阈值
    jerk_pass_thresh: float = 500.0 # jerk 阈值
```

### 8.2 参数调优指南

| 场景 | 建议调整 |
|------|----------|
| 规划失败率高 | ↑ `time_budget_per_segment`, ↑ `k_candidates` |
| 路径穿障 | ↓ `safe_margin` (如 0.3), ↑ `rrt_inflation` |
| 曲率过大 | ↓ `base_interp_dist`, ↓ `kappa_pass_thresh` |
| 计算太慢 | ↓ `max_iter_cap`, ↓ `time_budget_per_segment` |
| 路径太平滑 | ↑ `hdi_k_risk`, ↑ `via_cap_factor` |

## 9. 故障排查

### 9.1 编译失败：BOM 问题

**错误**：`MissingSectionHeaderError: File contains no section headers. file: 'setup.cfg', line: 1 '\ufeff[develop]'`

**解决**：
```bash
find ~/ta_hdi_ws -name "setup.cfg" -exec sed -i '1s/^\xef\xbb\xbf//' {} \;
find ~/ta_hdi_ws \( -name "*.msg" -o -name "*.srv" \) -exec sed -i '1s/^\xef\xbb\xbf//' {} \;
colcon build --symlink-install
```

### 9.2 Gazebo spawn 失败

**错误**：`Service /spawn_entity unavailable`

**原因**：gazebo_ros_factory 插件未正确加载

**解决**：
```bash
# 方法1: 手动 spawn
ros2 run gazebo_ros spawn_entity.py -database turtlebot3_waffle -entity waffle -x -2.0 -y -0.5 -z 0.01

# 方法2: 检查服务
ros2 service list | grep spawn
```

### 9.3 RViz 看不到机器人

**原因**：没有 `map` 坐标系（SLAM 未运行）

**解决**：
1. 将 RViz 的 **Fixed Frame** 从 `map` 改为 `odom`
2. 或启动 SLAM/AMCL 提供 map 坐标系

### 9.4 规划返回 NO_CANDIDATE

**原因**：RRT 在时间预算内未找到可行路径

**解决**：
1. 增加 `time_budget_per_segment`（如 5.0）
2. 增加 `max_iter_cap`
3. 检查 start/goal 是否在自由空间

### 9.5 规划返回 VERIFY_FAIL

**原因**：候选路径通过了 RRT 碰撞检测，但未通过验收

**解决**：
1. 降低 `safe_margin`（如 0.3）
2. 调整障碍物位置避免完全堵死
3. 增加候选池数量 `k_candidates`

## 附录 A：接口定义

### PlanPath2D.srv

```srv
# 请求
geometry_msgs/Pose2D start
geometry_msgs/Pose2D goal
ta_hdi_fmp_msgs/Obstacle2D[] obstacles

---
# 响应
bool success
nav_msgs/Path path
float32 min_dist
float32 kappa_p99
float32 jerk_rms
string status
```

### Obstacle2D.msg

```
# 常量
uint8 TYPE_CIRCLE = 1
uint8 TYPE_OBB = 2

# 字段
uint8 type       # 1=圆形, 2=矩形
float32 cx       # 中心 x
float32 cy       # 中心 y
float32 radius   # 圆形半径（TYPE_CIRCLE 用）
float32 width    # 矩形宽度（TYPE_OBB 用）
float32 height   # 矩形高度（TYPE_OBB 用）
float32 theta    # 旋转角度（TYPE_OBB 用）
```
