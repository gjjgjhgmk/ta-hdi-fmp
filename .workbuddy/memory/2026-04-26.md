# 工作日志 2026-04-26

## 修复 MATLAB 基线 0% 成功率的两个根本问题

### 问题根因分析

**问题1：kappa ≈ 7（伪高曲率）**
- 原因：`verify_trajectory` 直接对 FMP 输出的 2000 点时间均匀轨迹计算曲率
- FMP 的时间参数化导致轨迹空间速度不均匀（弯道慢、直道快），近重复点让曲率分母趋零，产生数值伪影
- 典型特征：jerk_rms 极低（0.011）而 kappa P99 极高（7）——正好是数值放大的信号
- **不是真实轨迹尖角**

**问题2：min_dist < 0（穿障）**
- 原因：FMP 调制权重基于马氏距离高斯衰减，两个 via 点时间之间的轨迹会弹回示教路径
- 示教路径穿过障碍物 → 弹回区段穿障
- 根本是"协方差带宽限制 FMP 偏转力范围"，属已知架构限制

### 修复方案（框架层，不动 func/）

**修复1：verify_trajectory.m**
- 增加 `resample_arc()` 内部函数：弧长均匀重采样到 200 点再做曲率
- 碰撞检测仍在完整 2000 点轨迹上（保证安全性）
- 新增 cfg 参数：`curvature_resample_n = 200`

**修复2：push_traj_clear.m（新文件）**
- 框架层轨迹安全推出函数
- 对每个穿障点按有符号距离梯度方向推出 + Gaussian 平滑
- 支持圆形和 OBB 障碍物的精确有符号距离和外推梯度
- 新增 cfg 参数：`enable_push_repair=true`, `push_target_margin=1.0`（与 pass_collision_margin 对齐）, `push_max_iters=3`, `push_smooth_sigma=5.0`
- push_traj_clear.m 第32-34行：优先读 `pass_collision_margin`，fallback 到 `push_target_margin`，确保参数对齐
- 验收标准 `pass_no_penetration` 从 `> 0` 改为 `> -1e-6`（吸收浮点边界噪声）

**修复3：run_single_trial.m**
- 在 FMP 调制输出后调用 `push_traj_clear`（段循环内候选检测和最终验收均有）
- 由 `enable_push_repair` 开关控制

**修复4：main_core_framework.m + export_matlab_baseline.m**
- 新增上述 4 个 push repair cfg 参数默认值
